# 무엇을 하는 공간인가요?
이곳은 스프링 부트를 자주 사용하다보니 부실해질 수 있는 스프링 서블릿, 필터, 리스너 등등에 대한 기초를 닦기위한 공간입니다.

하는김에 외장 톰캣을 다루는 방법, web.xml 등을 다루는 방법까지 직접 작업해보려 합니다.

# 1차 커밋
현재 모든 작업은 수작업 (gpt가 알려준 코드를 이해하며 따라 쳤습니다.)을 통해 이루어졌으며 
`C:\[제 폴더]\Tools\`에 제가 사용할 `Apache Tomcat`을 받아서 사용하였습니다.

이때 Tomcat의 버전은 현재 프로젝트의 서블릿 버전에 맞는 [톰캣 10.1.55버전](https://dlcdn.apache.org/tomcat/tomcat-10/v10.1.55/bin/apache-tomcat-10.1.55-windows-x64.zip)을 받아서 사용해주었습니다.

### 작업 과정
1. 우선 폴더를 직접 생성해주었습니다. (인텔리제이 무료판에서 사용하였습니다.)
   1. `README.md`를 가볍게 작성하였습니다.
   2. `simple-servlet-app`이라는 이름의 프로젝트를 생성하였으며 안쪽의 `./src/main/` 밑에 `/java/`소스 루트와 `/webapp`자원들의 경로를 만들어 두었습니다.
2. 이후 `simple-servlet-app/`에 `pom.xml`을 작성하여 기본적인 프로젝트 구조를 준비하였습니다. 
   이때 맨 위쪽의 `xmlns`, `xsi` 등이 뭔지 몰랏는데 찾아보니 스키마의 네임스페이스, xml 스키마 인터페이스 등을 바라보는 템플릿 같은 존재였습니다. 
   아래에는 현재 제 프로젝트의 메타데이터 (버전, war방식, url)과 빌드 방식, 환경 등을 작성하였으며 의존성으로는 `servlet-api:6.0.0`만을 사용한 순수 서블릿 프로젝트로 제작하였습니다. (scope는 provided로 톰캣의 서블릿 라이브러리를 사용하게 제작하였습니다.)
3. `mvn clean package`를 통해 프로젝트를 빌드하니 `simple-servlet-app/target`이 생성되었으며 내부에 `simple-servlet-app-1.0.0.war`이 존재하였습니다.
4. 생성된 `.war`파일을 Tomcat의 `C:\[제 폴더]\Tools\apache-tomcat-10.1.55\webapps`에 복사하니 `localhost:8080/simple-servlet-app-1.0.0/hello`를 가보니 `doGet()`에 만들어둔 대로 응답이 나가는 것을 확인할 수 있엇습니다.
### 후기
이제 앞으로 할일은 많이만 그동안 계속 확장만 하다가 오랜만에 서블릿을 복습할겸 톰캣도 직접 사용해보니 톰캣에 대한 이해도가 높아지는 경험을 할 수 있어서 좋았습니다.
종종 와서 기분전환으로 해당 작업을 하면 좋을 것 같습니다.
